# PRD — Hugh Interview Coach
**Version**: 1.0 (MVP)
**Status**: Ready for development
**Stack**: Next.js · Supabase · Claude API · ElevenLabs · Web Speech API

---

## 1. Problem & Vision

Preparing for technical interviews is difficult without a realistic practice environment. Reading Q&A lists is passive. Mock interviews with friends are hard to schedule. Hugh fills that gap: an AI-powered interview coach that simulates a real interview — voice, pressure, and all — for Data Engineering, Data Science, and ML Engineering roles.

The core experience: you enter a room, a persona greets you, asks you a question in their own voice, and listens to your answer. You get real feedback, not a generic tip sheet.

---

## 2. Target User (MVP)

Primary: Travis — a BI analyst actively transitioning into Data Engineering / Data Science roles, preparing for technical interviews independently.

Future: Any data professional preparing for technical interviews who wants structured, AI-powered practice with voice interaction.

---

## 3. MVP Scope

### In Scope
- User authentication (email + password)
- Room selection: Data Engineering, Data Science, ML Engineering
- Voice-driven interview loop (question → record → review → feedback)
- ElevenLabs persona voices (3 personas, randomly assigned per session)
- "Show Best Answer" option before answering
- Editable transcript after recording
- AI-generated feedback (context-aware: used best answer vs. own words)
- Session saved to Supabase (paused or completed)
- No-scroll desktop layout

### Out of Scope (v2)
- Voice analysis (fillers, pace, volume, confidence scoring)
- Performance dashboard and readiness score charts
- Mobile responsive layout
- Current events integration in questions
- Additional rooms beyond DE, DS, MLE
- AssemblyAI / Whisper STT upgrade

## 4. User Flows

### 4.1 Auth Flow
```
Visit hughapp.com
  → If not logged in: redirect to /login
  → Sign up (email + password) OR log in
  → Redirect to / (room selection)
```

### 4.2 Session Start Flow
```
Room selection page
  → User clicks "Enter Room" on a card (DE / DS / MLE)
  → Session created in Supabase (status: active)
  → Random persona assigned and stored in session
  → Navigate to /interview/[room]
  → First intro question generated (from hardcoded pool)
  → ElevenLabs TTS called server-side → audio ready
  → State: PLAYING_QUESTION (waveform animates, audio plays)
  → Audio ends → State: READY (buttons appear)
```

### 4.3 Answer Flow (Own Words)
```
State: READY
  → User clicks "I'm Ready"
  → State: RECORDING
  → Web Speech API starts, live transcript shown on screen
  → User speaks their answer
  → User clicks "Stop"
  → State: REVIEWING
  → Transcript shown in editable textarea
  → User edits if needed
  → User clicks "Submit Answer"
  → State: SUBMITTING
  → API: check-similarity (Claude judges if answer ≈ best answer)
  → API: generate-feedback (Claude generates feedback text)
  → API: tts (ElevenLabs generates feedback audio)
  → State: FEEDBACK
  → Waveform plays, feedback text shown below
  → Audio ends → "Next Question" and "Take a Break" buttons appear
```

### 4.4 Answer Flow (With Best Answer)
```
State: READY
  → User clicks "Show Best Answer"
  → Best answer text revealed below the question card
  → viewedBestAnswer flag set to true in session state
  → User clicks "I'm Ready" (same from here as 4.3)
  → On submit: similarity check runs
    → If Claude judges >90% alignment → usedBestAnswer = true
      → Feedback explains WHY this approach works
    → If Claude judges <90% alignment → usedBestAnswer = false
      → Feedback compares their answer to the suggested approach
```

### 4.5 Break Flow
```
"Take a Break" button available:
  - Bottom-left of question screen (state: READY)
  - After feedback (state: FEEDBACK)

On click:
  → Session status updated to 'paused' in Supabase
  → ended_at recorded
  → Redirect to / (room selection)
  → (v2: resume session from where left off)
```

### 4.6 Question Progression
```
Q1 (always): "Tell me about yourself and your background."
Q2 (always): Randomly selected from intro pool (experience, motivation, strengths)
Q3+: Dynamically generated by Claude for the selected room
       → Each question includes a generated best answer
       → Stored in Supabase questions table for the session
```

## 5. Screens

### Screen 1 — Login / Signup
- Clean centered card on dark background
- Tabs: "Sign In" / "Create Account"
- Fields: Email, Password (+ Confirm Password for signup)
- Supabase Auth handles session cookies

### Screen 2 — Room Selection (Landing)
- Full-viewport, no scroll
- Header: Hugh logo + user email + sign out link
- Tagline: *"Enter a room. Face the interview. Get better."*
- 3 cards in a row:
  - Data Engineering
  - Data Science
  - ML Engineering
- Each card: icon, title, short description, "Enter Room" button
- Design: dark navy (`#0F172A`), card surface (`#1E293B`), accent (`#38BDF8`)

### Screen 3 — Interview Room
Single full-viewport screen driven by interview state machine. Components shown/hidden based on state.

**Persistent across all states:**
- `PersonaBar` (top): avatar, persona name + role, room label, question counter

**State: PLAYING_QUESTION**
- `QuestionCard`: italic large question text, centered
- `WaveformPlayer`: animated waveform below the card (audio playing)
- No buttons visible

**State: READY**
- `QuestionCard`: italic large question text (stays visible)
- `BestAnswerPanel` (optional, appears if user clicks): best answer text revealed
- Two buttons: "Show Best Answer" (ghost) + "I'm Ready" (primary circular mic button)
- Bottom-left: "Take a Break" text link

**State: RECORDING**
- `QuestionCard`: smaller, moved to top of card
- `LiveTranscript`: real-time transcript text streaming below question
- Mic button: red pulsing state
- "Stop" button visible

**State: REVIEWING**
- `TranscriptEditor`: editable textarea pre-filled with transcript
- Helper text: *"Correct any transcription errors before submitting"*
- "Submit Answer" primary button
- "Re-record" text link

**State: SUBMITTING**
- Loading state: subtle spinner or skeleton
- Text: *"Hugh is reviewing your answer..."*

**State: FEEDBACK**
- Question shown small at top of card
- `WaveformPlayer`: animated waveform (feedback audio playing)
- Feedback text below waveform (bold first sentence, then detail)
- After audio ends: "Next Question" (primary) + "Take a Break" (ghost)

## 6. Data Model

### `sessions`
```sql
CREATE TABLE sessions (
  id            UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id       UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,
  room          TEXT CHECK (room IN ('data_engineering', 'data_science', 'ml_engineering')) NOT NULL,
  persona_id    TEXT NOT NULL,
  status        TEXT CHECK (status IN ('active', 'paused', 'completed')) DEFAULT 'active',
  question_count INTEGER DEFAULT 0,
  started_at    TIMESTAMPTZ DEFAULT NOW(),
  ended_at      TIMESTAMPTZ
);
```

### `questions`
```sql
CREATE TABLE questions (
  id             UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  session_id     UUID REFERENCES sessions(id) ON DELETE CASCADE NOT NULL,
  question_text  TEXT NOT NULL,
  best_answer    TEXT NOT NULL,
  question_type  TEXT CHECK (question_type IN ('intro', 'domain')) NOT NULL,
  order_index    INTEGER NOT NULL,
  asked_at       TIMESTAMPTZ DEFAULT NOW()
);
```

### `answers`
```sql
CREATE TABLE answers (
  id                  UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  question_id         UUID REFERENCES questions(id) ON DELETE CASCADE NOT NULL,
  transcript          TEXT NOT NULL,
  viewed_best_answer  BOOLEAN DEFAULT FALSE,
  used_best_answer    BOOLEAN DEFAULT FALSE,
  feedback_text       TEXT,
  submitted_at        TIMESTAMPTZ DEFAULT NOW()
);
```

### RLS Policies (all tables)
- Users can only SELECT, INSERT, UPDATE their own rows (`user_id = auth.uid()`)
- For `questions` and `answers`: join to `sessions` to verify ownership

## 7. API Routes

### `POST /api/interview/generate-question`
**Purpose**: Generate the next interview question + best answer for the session.

Request:
```json
{
  "room": "data_engineering",
  "questionType": "domain",
  "questionIndex": 3,
  "previousQuestions": ["..."]
}
```
Response:
```json
{
  "question": "Walk me through how you would design a fault-tolerant data pipeline.",
  "bestAnswer": "I would start by..."
}
```

### `POST /api/interview/generate-feedback`
**Purpose**: Generate feedback on the user's answer.

Request:
```json
{
  "question": "...",
  "bestAnswer": "...",
  "transcript": "...",
  "viewedBestAnswer": true,
  "usedBestAnswer": false
}
```
Response:
```json
{
  "feedback": "Good instinct identifying idempotency early. You missed the dead-letter queue pattern — that's what separates a resilient pipeline from a fragile one."
}
```

### `POST /api/interview/check-similarity`
**Purpose**: Claude judges whether user's answer aligns with the best answer.

Request:
```json
{
  "bestAnswer": "...",
  "transcript": "..."
}
```
Response:
```json
{
  "usedBestAnswer": true,
  "alignmentScore": 94
}
```

### `POST /api/interview/tts`
**Purpose**: Convert text to speech using ElevenLabs.

Request:
```json
{
  "text": "Tell me about a time you optimised a data pipeline.",
  "voiceId": "ELEVENLABS_VOICE_ID_1"
}
```
Response: Audio blob (MP3), streamed back to client.

## 9. Claude Prompt Strategy

### Question Generation
```
System: You are a senior interviewer for [room] roles at a [company type].
Generate one interview question appropriate for a mid-to-senior level candidate.
Avoid repeating these questions: [previousQuestions].
Return JSON: { "question": "...", "bestAnswer": "..." }
The best answer should be 3-5 sentences, structured clearly.
```

### Feedback Generation
```
System: You are a tough but fair interview coach.
The candidate was asked: [question]
The suggested best answer was: [bestAnswer]
The candidate answered: [transcript]
They [did / did not] view the suggested answer first.
Claude judged their answer as [used / not used] the suggested approach.

If usedBestAnswer = true: Explain why this approach is strong and what makes it the right answer. Acknowledge what they did well.
If usedBestAnswer = false: Give direct, constructive feedback. Highlight what they got right, what they missed, and what the best answer adds.
Keep feedback to 2-3 sentences. Start with a bold verdict phrase.
```

### Similarity Check
```
System: You are evaluating whether a candidate's interview answer matches the approach of a model answer.
Model answer: [bestAnswer]
Candidate answer: [transcript]
Does the candidate's answer capture at least 90% of the key points and approach from the model answer?
Respond with JSON only: { "usedBestAnswer": true/false, "alignmentScore": 0-100 }
```

## 10. Intro Question Pool (Hardcoded)

Always ask Q1. Randomly pick one from Q2 pool.

**Q1 (fixed):**
> "Thanks for coming in today. Let's start — tell me about yourself and your background."

**Q2 pool (pick 1):**
> "What drew you to data engineering / data science / ML engineering specifically?"
> "Walk me through your most technically challenging project."
> "How do you stay current with tools and developments in this space?"
> "What does your typical workflow look like when you start a new data project?"

## 11. Success Criteria (MVP)

- [ ] User can sign up, log in, and select a room
- [ ] A session starts with a randomly assigned persona
- [ ] Question is spoken aloud by the persona voice (ElevenLabs)
- [ ] Waveform animates during audio playback
- [ ] Buttons appear only after audio finishes
- [ ] User can optionally view the best answer before recording
- [ ] Web Speech API records and transcribes user's answer in real-time
- [ ] User can edit the transcript before submitting
- [ ] Claude judges similarity and generates context-aware feedback
- [ ] Feedback is spoken aloud by the same persona voice
- [ ] Session (questions + answers) is saved to Supabase
- [ ] User can take a break (session paused) at any time
- [ ] No screen requires scrolling on a 1080p desktop display
